## 1. UI 検知エンジン 『Project Tapper』

### ● 目的
UI ボタンの位置・形状・色を安定して検出し、実環境でもクリック成功率99%を実現することを目標にしました。

### ● 技術
* Python / OpenCV / NumPy / PyAutoGUI
* テンプレートマッチング
* 類似度スコア (`cv2.matchTemplate`)
* 閾値調整・座標計算
* システム負荷 (GPU/CPU) による誤差解析

### ● 主な成果
* クリック成功率: 最大99%
* 低バッテリー時に誤差増加のメカニズムを解明
* GPU負荷が高いほどフレーム遅延が発生し、認識ズレの原因となることを実証
* 色・形状・環境要因が一致率に与える影響を体系化

## 2. UI 感性検知テスト(色彩編)
~ AIが「正しい」だけでなく「心地よく感じる」反応を返すために ~
### UI 感性検知実験 (色彩認識の微差研究)

### 【概要】
AIの視覚認識モデルに対し、色彩・形状・配置の微差を提示して検出精度を比較。特にピンク(やや明度高) とレッド (中明度・高彩度)の弁別では、AIモデル間で反応差が顕著に現れた。

### ■ 色彩認識確率の比較データ
*※本データはサンプル値です。色彩心理学的要素を取り入れ、AIの「人間らしい感性模倣」の可能性を検証。*

| 形状カラー | 平均認識確率(%) | コメント |
| :--- | :---: | :--- |
| 白背景グレー枠 | 95% | コントラストが明確なため安定 |
| オレンジ | 92% | 高明度形状は認識が速い傾向 |
| レッド | 90% | 標準色として高認識率 |
| アンバー | 89% | 明度中程度では境界認識がやや低下 |
| ダークグレー | 87% | 背景との識別に見時間差あり |
| ピンク | 76% | 明度差による誤検知が発生 (AI間差あり) |

色彩は AI 認識に大きな影響を与えるため、複数色相・彩度・明度のUIボタンを比較しました。

### ● 主な発見
* ピンクやアンバーなど「明度差が小さい色」は誤検知が増える
* 背景とのコントラスト比が検出精度に直結
* 人が“見やすい”と感じる色と、AIが“認識しやすい”色は一致しない

### ● 結論
UIデザインとAI認識は別軸で成立するため、双方を考慮した最適設計が必要です。

## 3. UI 検知実験データとリズム最適化の比較分析
~ 「速さ」ではなく「自然さ」を再現することが目的である ~

### ① 環境別 UI 検知の成功率 (%)

| 実験環境 | 成功率(平均) | 誤検知率 | 備考 |
| :--- | :---: | :---: | :--- |
| 室内(昼光) | 98.7% | 1.1% | 最も安定した条件 |
| 室内(蛍光灯) | 97.2% | 1.8% | 明るさによる色差軽微 |
| 屋外(日陰) | 95.1% | 3.9% | 端末差による変化小 |
| 屋外(晴天) | 93.4% | 5.2% | 光反射の影響あり |

### ② 入力リズムパターンと誤差率の関係
以下は、入力間隔のパターンによる UI 検知の誤差率変化を示す。一定間隔よりも、人間的なゆらぎを持たせたランダムリズムの方が誤差率が低下した。

| リズムパターン | 平均誤差率 | 特徴 |
| :--- | :---: | :--- |
| ランダム±0.2秒 | 2.1% | 自然な挙動・安定 |
| リズム強調(1秒→1.3秒→1秒) | 2.8% | 周期感あり、安定性中程度 |
| 完全固定間隔(1秒) | 4.3% | 単調でエラー多め |

機械的な一定リズムのクリックは安定性が高いと思われがちですが、実験の結果、逆に誤差が増えることが判明しました。

### ● 結論
安定性 ＝ 機械的ではなく、“人間的”が最適となるケースが多い。
* UI 自動化に「人間の揺らぎ」を再現することが効果的

## 4. UI 自動操作システムの開発とマルチデバイス対応テスト
~ 再現性と信頼性を重視した自動化への挑戦 ~

### ● 概要
ユーザー操作を自動化し、業務効率を向上させる仕組みを開発。単なる動作確認にとどまらず、どの端末でも同様の結果を得られる再現性を重視。試行錯誤力と分析的思考を生かして取り組んだ。

### ● 課題
* UI操作が端末ごとに異なり、クリック座標がずれる。
* 解像度やスケーリング設定による誤動作が頻発。
* 一見単純な操作でも環境によって再現性が確保できない。

### ● 解決策
* 座標依存の操作をやめ、画像認識による相対位置検出を採用。
* 処理速度を上げるため、部分領域の認識を導入。
* iPhone/Android、Windows/Macなど複数端末で動作検証を実施。
* テスト結果を比較し、最も安定する手法を分析・記録。

### ● ターゲット設定と検出精度の工夫
操作対象として、アイコンを使用。微妙に異なる形状・色調の違いを識別するため、画像認識アルゴリズムのしきい値 (threshold) やマッチング精度を複数パターンで検証。結果、環境差による誤検出率を約10%以下に低減。

### ● 主な成果
* どの環境でも誤動作を起こさない安定したUI 自動化を実現。
* 自動処理の再現率を大幅に改善 (再現率約95%)。
* 自分で課題を発見→試行→分析→改善する習慣が身についた。

### ● 学び
* 失敗を繰り返しながら、「動く」 より 「再現できる」設計の重要性を実感。
* 多様な端末環境でテストすることで、ユーザー視点と開発者視点の両方を持つ大切さを学んだ。
* 今後はAIによるUI認識など、より高精度な自動化に挑戦したい。
